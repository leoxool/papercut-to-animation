# ArtClassroom 局域网测试指南

## 🎯 概述

本指南将帮助您在**局域网环境**中部署和测试 ArtClassroom 教室协作创作平台，模拟真实的课堂教学场景。

### 网络拓扑

```
                    教师电脑 (服务器)
                   ┌─────────────────┐
                   │  前端: 5173     │
                   │  WebSocket:8080 │
                   └────────┬────────┘
                            │ 局域网
              ┌─────────────┼─────────────┐
              │             │             │
         学生电脑1      学生电脑2      学生电脑3
        (客户端)       (客户端)       (客户端)
```

## 📋 环境准备

### 教师电脑（服务器）

**系统要求**
- macOS / Windows / Linux
- Node.js 18+ 已安装
- 稳定的局域网连接

**端口要求**
- 前端服务：`5173`
- WebSocket 服务：`8080`

### 学生电脑（客户端）

**系统要求**
- 任何现代浏览器（Chrome / Firefox / Safari / Edge）
- 连接到与教师电脑相同的局域网

## 🚀 部署步骤

### 第一步：教师电脑配置

#### 1.1 获取本机 IP 地址

**macOS / Linux**
```bash
ifconfig | grep "inet " | grep -v 127.0.0.1
```

**Windows**
```cmd
ipconfig
```

**示例输出**
```
inet 172.16.1.15 netmask 0xfffffc00 broadcast 172.16.3.255
```

**记录下这个 IP**，例如：`172.16.1.15`

#### 1.2 启动前端服务

在项目根目录执行：

```bash
npm run dev:lan
```

**预期输出**
```
  ➜  Local:   http://localhost:5173/
  ➜  Network: http://172.16.1.15:5173/
```

**重要**：确保看到 "Network" 提示，说明服务已监听所有网络接口。

#### 1.3 启动 WebSocket 服务器

打开**新的终端窗口**，执行：

```bash
cd server
npm install  # 首次运行需要安装依赖
npm start
```

**预期输出**
```
WebSocket server is running on port 8080
Accessible from all network interfaces
```

#### 1.4 验证服务

在教师电脑的浏览器中访问：
```
http://172.16.1.15:5173
```

确认可以正常访问。

### 第二步：学生电脑配置

#### 2.1 连接到局域网

确保学生电脑已连接到与教师电脑相同的 WiFi 网络。

#### 2.2 访问应用

在学生电脑的浏览器中访问：
```
http://172.16.1.15:5173
```

**将 `172.16.1.15` 替换为教师电脑的实际 IP**

#### 2.3 验证连接

- 页面正常加载
- 无网络错误
- 控制台无 CORS 错误

## 🔍 故障排查

### 问题 1：学生无法访问

**症状**
- 学生电脑访问 `http://172.16.1.15:5173` 失败
- 浏览器显示 "无法访问此网站"

**排查**
1. 确认教师电脑的 IP 是否正确
2. 确认前端服务已启动（终端显示 Network 地址）
3. 检查防火墙设置
4. 确认学生电脑与教师电脑在同一网络

**解决方案**
```bash
# 检查端口是否被占用
netstat -an | grep 5173

# 关闭防火墙（临时测试）
sudo ufw disable  # Linux
# 或在系统偏好设置中关闭 macOS 防火墙
```

### 问题 2：WebSocket 连接失败

**症状**
- 控制台显示 WebSocket 连接错误
- 学生提交素材后教师端收不到

**排查**
1. 确认 WebSocket 服务器已启动
2. 检查端口 8080 是否被占用
3. 查看浏览器控制台的错误信息

**解决方案**
```bash
# 检查 WebSocket 服务器
curl http://localhost:8080

# 重启 WebSocket 服务器
cd server
npm start
```

### 问题 3：页面加载但功能异常

**症状**
- 页面可以打开
- 但登录、注册等功能不工作
- 控制台有 JavaScript 错误

**排查**
1. 清除浏览器缓存
2. 禁用浏览器插件
3. 尝试无痕/隐私模式

**解决方案**
```javascript
// 在浏览器控制台中执行
localStorage.clear();
location.reload();
```

### 问题 4：素材上传失败

**症状**
- 学生可以登录和加入教室
- 但提交素材时失败

**排查**
1. 检查浏览器控制台的错误信息
2. 确认已选择要提交的素材
3. 检查 WebSocket 连接状态

### 问题 5：实时同步延迟

**症状**
- 学生提交后，教师端延迟几秒才收到
- 批准作品后，动画更新慢

**排查**
1. 检查网络延迟
2. 确认没有网络丢包
3. 检查设备性能

**解决方案**
- 关闭不必要的程序
- 确保网络稳定
- 重启服务

## 🔒 网络安全

### 防火墙配置

#### macOS
```bash
# 允许端口 5173 和 8080
sudo pfctl -f /etc/pf.conf
# 或在系统偏好设置 → 安全性与隐私 → 防火墙 中配置
```

#### Windows
```cmd
# 允许端口通过防火墙
netsh advfirewall firewall add rule name="ArtClassroom Frontend" dir=in action=allow protocol=TCP localport=5173
netsh advfirewall firewall add rule name="ArtClassroom WebSocket" dir=in action=allow protocol=TCP localport=8080
```

#### Linux
```bash
# 使用 ufw
sudo ufw allow 5173
sudo ufw allow 8080

# 或使用 iptables
sudo iptables -A INPUT -p tcp --dport 5173 -j ACCEPT
sudo iptables -A INPUT -p tcp --dport 8080 -j ACCEPT
```

### 网络隔离

**注意**：此配置仅用于教室内测试，不要在公网上暴露这些端口。

## 💡 最佳实践

### 教师端

1. **课前准备**
   - 提前 10 分钟启动服务
   - 测试网络连接

1. **网络要求**

1. **路由器配置**

   - 定期重启路由器
   - 监控网络负载

### 常见问题 FAQ

**Q: 可以连接但操作很慢？**
A: 检查网络带宽，建议使用有线连接或 5GHz WiFi。

**Q: 学生无法访问但教师可以？**
A: 检查防火墙设置和 IP 地址是否正确。

**Q: 可以访问但功能不正常？**
A: 清除浏览器缓存，使用无痕模式测试。

**Q: 多人同时在线会卡？**
A: 正常现象，建议分批测试或优化网络。
