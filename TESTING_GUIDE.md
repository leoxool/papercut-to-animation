# ArtClassroom 测试指南

## 🚀 快速开始

### 环境准备

#### 1. 启动前端服务
```bash
# 在项目根目录执行
npm install
npm run dev
```
前端服务将运行在: http://localhost:5173

#### 2. 启动 WebSocket 服务器
```bash
# 在另一个终端中执行
cd server
npm install
npm start
```
WebSocket 服务器将运行在: ws://localhost:8080

### 测试步骤

## 📝 测试场景一：教师端功能

### 步骤 1: 创建教师账户

1. 打开浏览器访问 http://localhost:5173
2. 点击右上角 "登录" 按钮
3. 点击 "注册新用户"
4. 输入用户名：`张老师`
5. 选择角色：`教师` ✅
6. 点击 "创建用户"

### 步骤 2: 创建教室

1. 登录后自动跳转到教师控制台
2. 点击 "创建教室" 按钮
3. 系统会生成一个 4 位数字代码（例如：1234）
4. 记录这个代码，学生需要用这个代码加入教室

### 步骤 3: 等待学生加入

- 右上角显示 "在线学生: 0"
- 等待学生使用代码加入教室

## 👨‍🎓 测试场景二：学生端功能

### 步骤 1: 创建学生账户

1. 打开 **新的浏览器窗口** 或 **隐身模式**
2. 访问 http://localhost:5173
3. 点击 "登录" → "注册新用户"
4. 输入用户名：`张三同学`
5. 选择角色：`学生` ✅
6. 点击 "创建用户"

### 步骤 2: 加入教室

1. 自动跳转到学生工作台
2. 看到 "加入教室" 对话框
3. 输入教师提供的 4 位数字代码（例如：1234）
4. 点击 "加入教室"

### 步骤 3: 采集素材

1. 在左侧素材库中，点击拍摄按钮或导入图片
2. 创建至少一个素材

### 步骤 4: 提交素材

1. 在素材库中选择一个素材（点击选中）
2. 点击 "提交给老师" 按钮
3. 确认提交成功

## 🔍 测试场景三：教师端审核

### 查看学生提交

1. 回到教师浏览器窗口
2. 在左侧 "学生作品" 面板中看到学生提交的作品
3. 作品状态显示为 "待审核"

### 审核操作

1. 点击学生作品查看详情
2. 点击 "通过" 按钮批准作品
3. 或者点击 "拒绝" 按钮拒绝作品

### 实时预览

1. 批准后，右侧 "实时动画预览" 中会显示学生的作品
2. 可以看到蝴蝶动画，使用学生提交的素材

## 🐛 问题排查

### WebSocket 连接失败

**症状**: 学生提交素材后，教师端没有收到

**排查**:
1. 检查 WebSocket 服务器是否启动：`http://localhost:8080`
2. 查看浏览器控制台是否有 WebSocket 错误
3. 确认端口 8080 没有被其他程序占用

### 素材提交失败

**症状**: 学生点击提交按钮没有反应

**排查**:
1. 检查是否已加入教室
2. 查看浏览器控制台错误信息
3. 确认选择了要提交的素材

### 角色切换不生效

**症状**: 登录后界面没有变化

**排查**:
1. 确认注册时选择了正确的角色
2. 刷新页面重新登录
3. 检查 localStorage 中的用户数据

### 动画不显示

**症状**: 教师批准作品后，动画区域是空的

**排查**:
1. 确认素材已批准（状态显示为 "已通过"）
2. 检查浏览器控制台是否有 Three.js 错误
3. 确认素材图片能正常加载

## 🔧 开发调试

### 查看 WebSocket 消息

在浏览器开发者工具中：
1. 打开 Network 选项卡
2. 筛选 WS (WebSocket) 连接
3. 查看发送和接收的消息

### 查看状态

在浏览器控制台中：
```javascript
// 查看当前用户
console.log(JSON.parse(localStorage.getItem('papercut_user')));

// 查看所有用户
console.log(JSON.parse(localStorage.getItem('papercut_all_users')));
```

### 重置数据

如果需要重置所有测试数据：
```javascript
// 在浏览器控制台中执行
localStorage.removeItem('papercut_user');
localStorage.removeItem('papercut_all_users');
location.reload();
```

## 🎓 教学场景模拟

### 完整课堂流程

1. **课前准备** (5 分钟)
   - 教师登录并创建教室
   - 生成教室代码

2. **学生加入** (5 分钟)
   - 学生依次登录
   - 输入代码加入教室

3. **创作环节** (20 分钟)
   - 学生采集和编辑素材
   - 学生提交作品给教师

4. **审核环节** (10 分钟)
   - 教师审核学生作品
   - 实时预览动画效果

5. **展示环节** (10 分钟)
   - 全班观看动画
   - 讨论和点评
